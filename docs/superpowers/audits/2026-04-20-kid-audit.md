## Findings

### 🔴 Broken

**1. Session cards on Accueil + Classes are not clickable.**
`UpcomingSessionList` renders each row as a plain `<div>`. Only the
"Rejoindre" button inside a joinable row is a link — ChevronRight is
just an icon. User taps anywhere on the card → nothing happens.
Affects both upcoming and past classes on /k/[id]/classes and the
"Cours d'aujourd'hui" section on /k/[id] home.

→ **Fixed**: wrapped the whole card in `<Link href={join_url}>`. The
joinable "Rejoindre" pill is now a `<span>` inside the card since the
parent link already owns the navigation. Chevron becomes purely
decorative. Target href resolves through `/k/[id]/class/[class_id]/room`
which redirects to `/session/[id]` — the one page that handles every
session state (waiting → countdown, ready → join, live → video room,
ended → recording).

### 🟡 Awkward

**2. Shortcut cards on Accueil use locale-less hrefs.**
`/k/[id]/classes` (no `/fr` prefix) vs sidebar links `/fr/k/[id]/classes`.
Middleware rewrites it correctly so it works, but inconsistent with the
rest of the app. Low priority — defer.

**3. "Past classes" link to the live-session room just like upcoming ones.**
A recorded class from 2 days ago links to `/k/[id]/class/[id]/room`
which redirects to `/session/[id]`. The session page then shows the
ENDED state with a "Revoir l'enregistrement" button — that works, but
it's two taps instead of one. Could link past classes directly to
`/api/recordings/[id]/play` for one-tap playback. Defer.

### 🟢 UX wins (not shipping this pass)

**4. Kid /courses empty state is friendly** ("Aucun cours. Demande à
tes parents de t'inscrire !") — good pattern, could extend to /classes.

**5. Kid /achievements shows a ribbon icon + "Tes succès apparaîtront
ici quand tu termineras des cours !" — nice tone, keep consistent
across empty states.
